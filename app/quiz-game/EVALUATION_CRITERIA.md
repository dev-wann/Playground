# 프론트엔드 과제 평가 가이드

## 공통 평가 항목

| 영역 | 평가 포인트 |
|---|---|
| 코드 품질 | 컴포넌트 분리 기준이 합리적인가, 관심사 분리가 되어 있는가 |
| 커스텀 훅 | 반복 로직(타이머, 폼 검증 등)을 훅으로 추출했는가 |
| 타입 안전성 | TypeScript 사용 시 any 남용 없이 적절한 타입 정의를 했는가 |
| 렌더링 최적화 | 불필요한 리렌더링을 인지하고 memo/useMemo/useCallback을 적재적소에 사용했는가 |
| UX 디테일 | 로딩, 에러, 비활성 상태 등 엣지 케이스를 빠뜨리지 않았는가 |

## 커스텀 퀴즈 게임

### 폼 평가

| 항목 | 구체적으로 보는 것 | 왜 중요한가 |
|---|---|---|
| 다중 선택 관리 | 카테고리 복수 선택/해제가 정확한가, 최소 1개 제한을 어떻게 처리했는가 | 배열 상태 조작 |
| 동적 제약 조건 | 선택된 카테고리 수에 따라 최대 문제 수가 변하고, 이를 검증하는가 | 폼 필드 간 의존성 처리 |
| 슬라이더 + 표시 | range input의 값이 실시간으로 레이블에 반영되는가 | 제어 컴포넌트 기본기 |
| 조건부 기능 전달 | 힌트 토글 ON/OFF가 게임 페이지의 힌트 버튼 존재 여부에 정확히 연결되는가 | 설정 → 게임 간 상태 전달 |

### 상태 관리 평가

| 항목 | 구체적으로 보는 것 | 왜 중요한가 |
|---|---|---|
| 멀티 스텝 흐름 | 문제 풀기 → 정답 확인 → 다음 문제 → 최종 결과, 각 단계 전환이 매끄러운가 | 다단계 상태 전이 |
| 타이머 연동 | 답 선택 시 타이머 정지, 다음 문제 시 리셋, 타임아웃 시 자동 오답 처리 | 비동기 상태와 UI 동기화 |
| 타이머-타임아웃 레이스 컨디션 | 타이머 만료와 사용자 클릭이 동시에 발생할 때 결과가 중복 기록되지 않는가 | 비동기 경쟁 상태 방어 |
| 누적 데이터 | 각 문제의 정답 여부, 획득 점수, 힌트 사용 여부를 누적 저장해 결과에 반영하는가 | 히스토리 상태 관리 |
| 점수 계산 | 난이도 가중치 + 시간 보너스 + 힌트 감점이 정확하게 계산되는가 | 복합 파생 값 처리 |
| 힌트 시스템 | 오답 2개 제거가 정답을 건드리지 않는가, 문제당 1회 제한이 지켜지는가 | 조건부 상태 변경 |
| 더블 클릭 방어 | 답 선택 직후 즉시 상태를 잠가 빠른 연타로 중복 제출이 발생하지 않는가 | 동시성 버그 방어 |

### UX / 접근성 평가

| 항목 | 구체적으로 보는 것 | 왜 중요한가 |
|---|---|---|
| 키보드 조작 | 숫자키(1-4)로 답 선택, Enter/Space로 다음 문제 진행이 가능한가 | 키보드 접근성 |
| 키보드-상태 연동 | 답 선택 후 숫자키 비활성, 힌트로 숨긴 선택지의 키 입력 무시 등 상태에 따라 키보드 입력이 올바르게 제한되는가 | 상태 기반 입력 제어 |
| 모달 중 입력 차단 | 나가기 모달이 열린 동안 타이머가 멈추고 키보드 입력이 무시되는가 | 모달 UX 완성도 |
| 타이머 시각 피드백 | 남은 시간에 따라 색상이 변하는가 (초록 → 노랑 → 빨강) | 시간 압박의 직관적 전달 |
| 답변 후 UI 잠금 | 답 선택 후 정답/오답 표시와 함께 선택지가 비활성화되는가 | 상태 기반 UI 피드백 |
